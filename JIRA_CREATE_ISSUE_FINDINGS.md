# Изучение страницы Jira Create Issue (живая страница)

Источник: снимок через **user-playwright** MCP после авторизации на https://jira.bcs.ru/secure/CreateIssue.jspa

## Общая структура

- **Заголовок**: «Создать задачу»
- **Табы**: Main (выбран), Value, Design, Planning, WSJF
- **Поля на вкладке Main**: Проект (My broker), Тип задачи (Idea), Приоритет, Исполнитель, **Тема**, **Описание**, далее кастомные поля (Стримы инсайт, CheckList и т.д.)

## Поле «Тема» (Summary)

- Обычное поле ввода (textbox), placeholder «Название темы»
- В прототипе предзаполняем значением: «Показывать инвестиционные идеи после совершения сделки»

## Поле «Описание» — особенности (живая Jira)

1. **Два режима отображения**
   - **Визуальный** (кнопка «Визуальный» в состоянии pressed): пользователь видит и редактирует контент в **iframe** (Rich Text Area). Контент отображается как отрендеренный HTML: заголовки (h3), панели с цветными заголовками (Сейчас, Гипотеза 1, UserStory и т.д.), параграфы, выделение жирным.
   - **Текст**: пользователь видит и редактирует **textarea** с исходной wiki-разметкой Jira (h3., {panel:...}, *...* и т.д.).

2. **Один источник истины**
   - Содержимое хранится как **wiki-разметка**. В снимке доступности:
     - `textbox "Описание"` содержит полную строку wiki-разметки;
     - `iframe` (роль application, «Описание, Rich Text Area. Press ALT-0 for help.») содержит тот же контент в виде отрендеренного HTML (heading, generic, paragraph, strong).

3. **Тулбар над полем (в визуальном режиме)**
   - Стиль (dropdown), Полужирный (⌘+B), Курсив (⌘+I), Подчеркнуть (⌘+U)
   - Цвет текста, Больше цветов, Дополнительные опции форматирования
   - Ссылка (⌘+K), Вложение
   - Маркированный список, Нумерованный список
   - Значки (эмодзи), Другое
   - Кнопка «Скрыть панель инструментов»

4. **Переключатель режимов**
   - Две кнопки: «Визуальный» (активна) и «Текст»
   - Статус: «Визуальный режим»

5. **Кнопки Отменить / Вернуть**
   - Под тулбаром (для undo/redo)

## Прототип jira-idea-edit.html

- Собран на основе **сохранённой** страницы Jira («Сохранить как» → «Веб-страница, полностью»). В сохранённой версии редактор описание — это **wiki-редактор**: одна **textarea** с wiki-разметкой и тулбар (Стиль, жирный, курсив, ссылки, панели и т.д.) в виде dropdown’ов.
- В живой Jira при выборе «Визуальный» редактор показывает **iframe** с WYSIWYG; при «Текст» — ту же textarea с разметкой.
- В прототипе: **редактируемая textarea** с предзаполненной wiki-разметкой; скрипт при загрузке копирует атрибут `value` в `textarea.value`, чтобы поле было и видимым, и редактируемым. Это эквивалент режима «Текст» в живой Jira.
- Ресурсы (CSS/JS) подключаются из `../../Создать задачу - Jira_files/` (папка в Загрузках рядом с исходным сохранённым HTML).

## Визуальный редактор по wiki-разметке (прототип)

Чтобы в прототипе показывать разметку как во вкладке «Визуально» и дать пользователю её редактировать:

- **`jira-wiki-render.js`** — парсер Jira wiki → HTML (подмножество: `h1.–h6.`, `*жирный*`, `{panel:title=...|...}...{panel}`).
- **`jira-wiki-visual.css`** — стили панелей и contenteditable-области.
- **`jira-visual-editor.html`** — страница с полем «Тема» и визуальным полем «Описание» (contenteditable). Кнопка «Дополнить идею» на доске ведёт сюда.

**Как передать свою разметку на вход:**

1. **По умолчанию** — в странице вставлен блок `<script type="text/plain" id="wiki-source">...</script>` с разметкой. Замените его содержимое или подставьте разметку при сборке/отдаче страницы.
2. **Через глобальную переменную** — перед загрузкой `jira-wiki-render.js` задать `window.JIRA_WIKI_INITIAL = "h3. Заголовок\n...";`. Тогда будет использована она, а не `#wiki-source`.

Пользователь видит отрендеренный результат (заголовки, цветные панели, жирный текст) и может редактировать его в contenteditable.

## Рекомендация для юзертеста

Для сценария «как пользователь взаимодействует с полем ввода с предзаполненным текстом» текущий прототип (редактируемая textarea с wiki-разметкой) достаточен: пользователь может скроллить, выделять, править текст. Если нужно максимально приблизить к живой Jira, потребуется либо переключатель «Визуальный/Текст» с отображением только текстового режима, либо встраивание отдельного WYSIWYG-редактора с рендером wiki (сложнее).
